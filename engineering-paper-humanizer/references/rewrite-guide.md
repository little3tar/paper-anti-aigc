# 重写指南：工程类学术文本去 AI 化

> 本文件是 engineering-paper-humanizer 的核心改写策略。包含五大类规则、灵魂注入指南和 LaTeX 保护红线。
>
> **注意**：AI 敏感词替换字典和快速检查清单已移至 `scripts/rules.json`，运行 `python scripts/check_aigc.py <file>` 即可获取完整检测。

---

## Part 1：五大核心规则分类

> **重构说明**：原十五个维度按工程写作的实际问题类型重新归为五大类，每类包含 3-4 条具体规则。规则顺序不再固定，可按需交叉引用。

### 一、内容逻辑类（Content & Logic）

工程写作的核心是清晰、准确、可追溯的技术内容表达。

#### 规则 L1：重塑"僵化逻辑"与"过渡句式"

**敏感词**：虽然...但是、因此、然而、此外、综上所述、总而言之、由此可见、不难发现

**操作**：

- 根除"虽然...但是"模板 → 直接客观陈述双方侧重点
- 克制连接词泛滥 → 强制删除至少 50% 的强连接词（因此、然而、此外），用工况条件推导天然结论

#### 规则 L2：重构"文献综述"起手式

**敏感词**：现有研究表明、大量文献指出、国内外学者普遍认为、国内起步较晚、研究尚处于起步阶段

**操作**：

- ❌ "现有研究表明..." → ✅ "某学者指出..."或直接陈述结论
- ❌ "国内起步较晚..." → ✅ 直接陈述"近年来，国内学者围绕XXX展开了针对性研究"

#### 规则 L3：消除"模糊归因"与"虚假权威"

**敏感词**：众所周知、业内公认、专家认为、有关研究表明、大量实验证明、普遍认为、据报道

**操作**：

- 强制具体化归因 → 坚决剔除"众所周知"、"业内公认"、"专家认为"。必须精确到具体的参考文献编号或作者姓名
- 清理虚假权威 → 删除"大量实验证明"（若无具体实验数据支撑）

#### 规则 L4：消除"提纲式挑战"部分

**操作**：

- ❌ "尽管取得了进展，但系统仍面临若干挑战..." → ✅ 直接陈述具体的技术瓶颈（如"高频噪声干扰仍未完全滤除"）

#### 内容逻辑类改写示例

**输入（AI 味道）：**

```latex
现有研究表明，振荡截割技术在硬岩破碎领域具有十分重要的工程应用价值。
尽管取得了一定的进展，但该领域仍然面临若干挑战，包括刀具磨损、粉尘控制和火花灾害隐患。
```

**输出（人性化）：**

```latex
Hood 等\\cite{hood2005}2005年提出的振荡截割概念利用岩石抗拉强度仅为抗压的$1/10$这一特性，
迫使岩石沿拉伸面破裂。但镜型截齿在UCS$>100\\,\\mathrm{MPa}$工况下磨损过快——
这是MC51转向碟盘刀具的直接原因。
```

**变更说明：** 删除“现有研究表明”模糊归因，改为具体学者引用；删除“具有十分重要的工程应用价值”空泛声明；将“面临若干挑战”改为具体技术事实。

### 二、语言表达类（Language & Expression）

从词汇选择到句式节奏，消除 AI 生成文本的语言模式。

#### 规则 E1：降解"宏大叙事"与"空泛表达"

**敏感词**：应运而生、为了能够有效解决、重大难题、彻底解决、卓越性能、显著优越性、具有十分重要的工程应用价值、极大地提升、有效地改善、作为...的关键

**操作**：

- 砍掉起手式废话 → 坚决删除"为了能够有效解决...重大难题"、"应运而生"。直接抛出技术动作或工况事实
- 降级绝对化词汇 → 将"彻底解决"、"卓越性能"降级为平实的"有效改善"、"有效降低"
- 去除空泛呼吁 → 禁用"具有十分重要的工程应用价值"。改为具体学术贡献

#### 规则 E2：替换"AI高频敏感词组"

**操作**：

- 运行 `check_aigc.py` 获取完整敏感词替换字典
- 核心原则：用具体技术描述替代空洞套话

#### 规则 E3：提振突发性（Burstiness）与扰动困惑度（Perplexity）

**操作**：

- 提振突发性 → 主动使用 3~5 字极简短句，紧接 20+ 字参数密集长句。人为制造大开大合"顿挫感"
- 扰动困惑度 → 刻意保留工程粗糙感词汇（"受限于井下防爆空间定额，妥协保留了..."、"被迫采用"）

#### 规则 E4：根治"同义词循环"与"虚假范围"

**敏感词**：在某种程度上、从...到...（虚假范围）、一定程度上、某种意义上

**操作**：

- 锁定核心术语 → 同一物理量或设备部件必须使用完全相同的称谓，严禁刻意换词
- 拆解虚假范围词 → 删除"在某种程度上"、"从 X 到 Y"（当 X 和 Y 不在有意义的尺度上时）

#### 规则 E5：削减"填充短语"与"过度限定"

**敏感词**：为了实现这一目标、值得注意的是、总的来说、可以说、基本上、似乎、在一定程度上、具有...的能力

**操作**：

- 修剪填充废话 → 删除"为了实现这一目标"、"值得注意的是"、"总的来说"
- 压缩过度限定 → 删除"可以说"、"基本上"、"似乎"。工程结论应当是确定性的
- 拒绝通用积极结尾 → 删除段末无实际意义的乐观展望（如"具有广阔的应用前景"）

#### 语言表达类改写示例

**输入（AI 味道）：**

```latex
此外，模糊控制策略的引入显著地提升了系统性能，具有广阔的应用前景。
值得注意的是，该算法不仅能够有效地改善响应速度，而且在一定程度上降低了超调量。
```

**输出（人性化）：**

```latex
引入模糊规则后，超调量从$12.3\\%$压至$4.7\\%$，调节时间缩短约$40\\%$。
代价是离线整定约40条模糊规则，现场调参耗时两个工作日。
```

**变更说明：** 删除“此外”“值得注意的是”“不仅…而且…”“在一定程度上”“具有广阔的应用前景”；用具体参数替代“显著地提升”；注入工程妥协感（调参代价）。

#### 规则 E6：避免使用"是"（系动词回避）

**操作**：

- ❌ "该算法作为提升精度的关键体现" → ✅ "该算法提升了精度"

#### 规则 E7：打破"否定式排比"

**操作**：

- ❌ "这不仅仅是一次算法升级，而是控制理念的革命" → ✅ "该算法升级改变了控制策略"

#### 规则 E8：消除破折号滥用与连接词暴露

**操作**：

- 删除揭示性破折号 → 删除用于揭示或解释的破折号，改用逗号或句号
- 检查连接词替代 → AI 有时用破折号替代"即"、"也就是说"等连接词，需检查上下文是否自然
- 规范破折号使用 → 中文破折号应使用全角"——"，仅用于语义转折或强调，不用于常规连接

### 三、格式结构类（Format & Structure）

LaTeX 格式规范与文档结构优化，消除 AI 排版模式。

#### 规则 F1：打破"机械结构"与"流水线排版"

**敏感词**：本章将针对...开展工作、本节主要内容包括、下面将分别介绍、具体如下、如图所示

**操作**：

- 抹除章节预告 → 禁止在段首写"本章将针对...开展工作"
- 消灭八股列表 → 禁止使用"名词+冒号+长篇解释"的清单体。融合成长短不一的连贯段落
- 打破看图说话 → 禁止在插入图片后紧跟独立解释段。图片引用自然融入主正文

#### 规则 F2：修复"标点暴露"与"LaTeX格式漏洞"

**操作**：

- 引用位置红线 → `\cite{...}` 紧贴被引文字，禁止前加空格，标记在句号/逗号内侧
- 引号规范 → 强制中文双引号 "”，禁止使用弯引号（""）
- 括号清理 → 严禁堆砌连续括号，解释性文字直接化入正文

#### 规则 F3：抑制"过度修饰"与"格式暴露"

**敏感词**：实现着、体现着、展示着、彰显着（-ing式虚假深度）

**操作**：

- 清空表情与特殊装饰 → 严禁出现 Emoji 或机械地用粗体强调短语。粗体仅限首次定义术语使用
- 纠正"着"字虚假深度 → 删除模仿英文 -ing 句式的"实现着..."、"体现着..."
- 内联标题垂直列表 → 禁止输出 `\item \textbf{名词：} 解释` 格式的列表

#### 格式结构类改写示例

**输入（AI 味道）：**

```latex
本章将针对液压系统开展设计工作。具体如下：
\begin{itemize}
  \item \textbf{泵源选型：} 采用负载敏感变量泵，根据负载压力自动调节流量。
  \item \textbf{阀组配置：} 采用比例方向阀实现流量精确控制。
\end{itemize}
```

**输出（人性化）：**

```latex
泵源选用负载敏感变量泵，输出压力始终跟随最高负载压力加上$\Delta P_{\mathrm{LS}}$，
多余流量不再经溢流阀发热。三缸各配一套比例方向阀，独立调节流量与方向。
```

**变更说明：** 删除"本章将针对…开展工作"章节预告；将 `\item \textbf{...}` 清单体融合成连贯段落；补充了 $\Delta P_{\mathrm{LS}}$ 等具体参数。

### 四、工程特质类（Engineering Characteristics）

注入真实的工程思维、妥协与专业判断。

#### 规则 T1：展现工程妥协与真实约束

**操作**：

- 承认技术限制 → 明确说明"虽然提升了精度，但代价是算力开销增加了 30%"
- 体现硬件约束 → 描述"受限于井下防爆空间定额，妥协保留了..."、"被迫采用"等现实决策
- 展示参数权衡 → 用具体数字说明性能提升与代价的平衡

#### 规则 T2：定量参数碾压

**操作**：

- 替换"大幅"、"极大地" → 改为具体百分比或数值（如 $12.3\%$、$3.8\,\mathrm{s}$）
- 补充基准对比 → 说明"比传统方案提升 37%"中的传统方案具体指标
- 保留单位与精度 → 使用正确的工程单位（MPa、kN、Hz等）和合理的有效数字

#### 工程特质类改写示例

**输入（AI 味道）：**

```latex
3-RPS并联机构极大地提升了截割系统的性能，彻底解决了传统悬臂式方案的困境。
```

**输出（人性化）：**

```latex
3-RPS并联机构将空间位姿调整与高频振荡破岩解耦。但三条支链非对称协调伸缩的控制难度
明显高于串联悬臂——单缸跟踪误差需压制在$\pm 0.5\,\mathrm{mm}$以内，否则末端轨迹偏移经雅可比
矩阵放大后将超出刀具容差。
```

**变更说明：** 删除"极大地提升""彻底解决"绝对化词汇；注入工程妥协（"控制难度明显高于"）和定量约束（$\pm 0.5\,\mathrm{mm}$）。

### 五、交流痕迹类（Communication Traces）

清除聊天机器人对话残留与 AI 身份暴露。

#### 规则 C1：清除"协作交流痕迹"与"AI身份残留"

**敏感词**：希望这些信息对您有帮助、好问题、您说得完全正确、请告诉我、截至目前、根据我最后的训练更新、虽然具体细节有限

**操作**：

- 剔除对话口癖 → 彻底删除"希望这些信息对您有帮助"、"好问题"、"您说得完全正确"
- 抹除知识时效免责 → 删除"截至目前"、"根据我最后的训练更新"
- 消除谄媚语气 → 删除过于积极、讨好的语言

#### 规则 C2：避免教学式过度解释

**操作**：

- 删除基础定义 → 如"液压系统是一种利用液体压力传递动力的系统..."（读者是专业人士）
- 简化技术介绍 → 直接使用专业术语，不附加解释性括号
- 信任读者背景 → 假设读者具备领域基础知识，不进行入门级说明

#### 交流痕迹类改写示例

**输入（AI 味道）：**

```latex
电液伺服系统（即一种利用液压油作为介质的伺服驱动装置）具有响应快、出力大、控制精度高的特点。
它在工业自动化中具有广阔的应用前景。
```

**输出（人性化）：**

```latex
截割臂三缸均采用电液比例伺服驱动，额定压力$31.5\,\mathrm{MPa}$，
行程$\pm 150\,\mathrm{mm}$。
```

**变更说明：** 删除"（即一种…）"教学式解释（读者是专业人士）；删除"响应快、出力大、控制精度高"三段式套话；删除"具有广阔的应用前景"；补充具体参数。

## Part 2：个性与灵魂（Soul & Personality）

避免 AI 模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。好的学术/工程写作背后有一个真实的工程师。

### 缺乏灵魂的写作迹象（即使技术上"干净"）

- 每个句子长度和结构都相同（缺乏节奏）
- 只有中立报道，没有工程视角的取舍和判断
- 不承认不确定性、复杂工况或现实妥协
- 读起来像产品新闻稿或维基百科文章

### 如何增加工程语调（注入人味）

1. **展现工程妥协（承认复杂性）**：真实的工程没有完美的方案。"虽然提升了精度，但代价是算力开销增加了 30%" 胜过单方面的"该方法表现出显著优越性"
2. **变化节奏（突发性）**：短促有力的定性结论。然后是参数密集的定量长句。混合使用
3. **定量参数碾压**：用具体的数字（如 $12.3\%$、$3.8\,\mathrm{s}$）替代"大幅"、"极大地"等虚词
4. **允许一些"毛刺"**：完美的结构感觉像算法。保留一些工程实践中的"被迫采用"、"受限于硬件"等真实困境描述

## Part 3：LaTeX 绝对保护红线

### 禁止全局替换

禁止使用 replaceAll 修改 LaTeX 命令。缺失反斜杠时（如 `includegraphics`），只在当前单词前补全 `\`。

### 排版环境绝对保护

绝不修改、删除或翻译任何以 `\` 开头的命令和环境。特别注意：

- `ctexart` 中文字体设置命令（`\zihao`、`\songti` 等）
- `\begin{}`/`\end{}` 环境声明
- `\label{}`、`\ref{}`、`\includegraphics{}` 等交叉引用命令

### 数学体系免疫

`equation`、`align` 等数学公式环境块内部的字母推导、等式演算 **绝对禁止修改**。

### 禁止内部换行

- 严禁将 `\cite{...}` 或连同前后符号拆分到多行
- 严禁在 `\label{...}` 或 `\includegraphics{...}` 的花括号内部换行
- 百分号必须输出为 `\%`

## 使用说明

1. **改写时参考**：按五大类规则逐段扫描，结合灵魂注入指南
2. **敏感词检测**：运行 `python scripts/check_aigc.py <file>` 获取完整敏感词列表和位置
3. **自检**：运行 `check_aigc.py` 即为自检，无需手动对照检查清单
4. **格式支持**：
   - LaTeX 文件：`python scripts/check_aigc.py paper.tex`
   - Markdown 文件：`python scripts/check_aigc.py paper.md --format markdown`
   - 纯文本文件：`python scripts/check_aigc.py draft.txt --format plain`
