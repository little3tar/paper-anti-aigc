## Part 1：十五大核心维度

### 维度一：降解"宏大叙事"与"空泛表达"

**需要注意的词汇**：应运而生、为了能够有效解决、重大难题、彻底解决、卓越性能、显著优越性、具有十分重要的工程应用价值、极大地提升、有效地改善、作为...的关键

**问题**：真实的工程师行文客观克制，绝不使用口号式的虚词填补字数。

- **砍掉起手式废话**：坚决删除"为了能够有效解决...重大难题"、"应运而生"。直接抛出技术动作或工况事实。
- **降级绝对化词汇**：将"彻底解决"、"卓越性能"降级为平实的"有效改善"、"显著降低"。
- **去除空泛呼吁**：禁用"具有十分重要的工程应用价值"。改为具体学术贡献。

**改写前**：

```latex
为了能够有效解决深部硬岩截割的重大难题，本系统应运而生，采用了先进的振荡截割技术，彻底解决了传统方法的局限性，具有十分重要的工程应用价值和卓越性能。
```

**改写后**：

```latex
针对深部硬岩（UCS > 100 MPa）截割时传统镐型截齿磨损快的问题，本系统采用振荡截割技术，将截割反力从 120 kN 降至 12 kN，刀具寿命延长约 3 倍。
```

### 维度二：打破"机械结构"与"流水线排版"

**需要注意的词汇**：本章将针对...开展工作、本节主要内容包括、下面将分别介绍、具体如下、如图所示

**问题**：AI 极度痴迷于对称和预告，必须彻底打碎这种规律性。

- **抹除章节预告**：禁止在段首写"本章将针对...开展工作"。
- **消灭八股列表**：禁止使用"名词+冒号+长篇解释"的清单体。融合成长短不一的连贯段落。
- **打破看图说话**：禁止在插入图片后紧跟独立解释段。图片引用自然融入主正文。

**改写前**：

```latex
本章将针对液压系统开展详细设计工作，主要内容包括：
\begin{itemize}
\item \textbf{泵站选型：}根据系统流量和压力需求，选择合适的液压泵。
\item \textbf{管路设计：}计算管径，确保压降在允许范围内。
\item \textbf{冷却系统：}设计散热器，防止油温过高。
\end{itemize}
如图~\ref{fig:pump}所示，液压泵安装在底盘左侧。该布局便于维护。
```

**改写后**：

```latex
泵站选用 A10VSO140 变量柱塞泵（排量 140 mL/r，额定压力 31.5 MPa），实测最大流量 210 L/min 时压降 0.8 MPa，满足三缸同时动作工况。主管路采用 DN32 无缝钢管，冷却器布置在泵站出口（图~\ref{fig:pump}），强制风冷将油温控制在 65°C 以下。
```

### 维度三：重塑"僵化逻辑"与"过渡句式"

**需要注意的词汇**：虽然...但是、因此、然而、此外、综上所述、总而言之、由此可见、不难发现

**问题**：用真实工程的物理因果关系推动行文。

- **根除"虽然...但是"模板**：直接客观陈述双方侧重点。
- **克制连接词泛滥**：强制删除至少 50% 的强连接词（因此、然而、此外），用工况条件推导天然结论。

**改写前**：

```latex
虽然模糊PID控制能够提升系统响应速度，但是其规则库整定复杂。因此，本文采用了自适应模糊PID方案。此外，还引入了卡尔曼滤波抑制噪声。综上所述，该方案具有良好的控制性能。
```

**改写后**：

```latex
模糊PID将超调量从 12.3\% 压至 4.7\%，代价是离线整定 40 条规则耗时两个工作日。卡尔曼滤波将位移传感器噪声从 ±0.5 mm 降至 ±0.1 mm，跟踪误差减小 18.94\%。
```

### 维度四：修复"标点暴露"与"LaTeX格式漏洞"

**需要注意的词汇**：（检查格式而非词汇）

**问题**：标点和LaTeX命令的位置错误会暴露AI生成痕迹。

- **引用位置红线**：`\cite{...}` 紧贴被引文字，禁止前加空格，标记在句号/逗号内侧。
- **引号规范**：强制中文双引号 “”，禁止使用弯引号（""）。
- **括号清理**：严禁堆砌连续括号，解释性文字直接化入正文。

**改写前**：

```latex
Hood等人提出了振荡截割概念 \cite{hood2006}。该方法"利用岩石抗拉强度低的特点"（仅为抗压强度的1/10）（通过高频振荡实现）。
```

**改写后**：

```latex
Hood等人\cite{hood2006}提出振荡截割概念，利用岩石抗拉强度仅为抗压强度 1/10 的特点，通过高频振荡迫使岩石沿拉伸面破裂。
```

### 维度五：替换"AI高频敏感词组"

**需要注意的词汇**：详细替换表见本文 Part 3（双重困境、协同难题、旨在解决、突破技术瓶颈、取得了一定进展、此外、至关重要、深入探讨、强调、复杂性、格局、证明、充满活力的、著名的）。

**问题**：这些词在 2023 年后的 AI 生成文本中出现频率异常高。

**改写前**：

```latex
此外，本文深入探讨了液压系统的复杂性问题，旨在解决双重困境，突破了关键技术瓶颈，取得了一定进展，这对于整个行业格局至关重要，充分证明了该方案的先进性。
```

**改写后**：

```latex
液压系统存在流量-压力耦合与死区非线性两个核心问题。本文通过死区逆模型补偿将静态误差从 1.2 mm 降至 0.4 mm，压力前馈将动态响应带宽从 5 Hz 提升至 9 Hz。
```

### 维度六：重构"文献综述"起手式

**需要注意的词汇**：现有研究表明、大量文献指出、国内外学者普遍认为、国内起步较晚、研究尚处于起步阶段

**问题**：模糊的文献综述起手式缺乏具体归因。

- ❌ "现有研究表明..." → ✅ "某学者指出..."或直接陈述结论
- ❌ "国内起步较晚..." → ✅ 直接陈述"近年来，国内学者围绕XXX展开了针对性研究"

**改写前**：

```latex
现有研究表明，振荡截割技术能够有效降低截割阻力。国内起步较晚，但近年来也取得了一定进展。大量文献指出该技术具有广阔的应用前景。
```

**改写后**：

```latex
Erarslan\cite{erarslan2014}在DCD实验中发现，振荡频率20~45 Hz时法向力降低25\%~30\%。国内方面，刘春生\cite{liu2018}通过齿数优化将径向载荷峰值削减18\%。
```

### 维度七：提振突发性（Burstiness）与扰动困惑度（Perplexity）

**需要注意的词汇**：（关注句式节奏而非特定词汇）

**问题**：AI生成的文本句子长度均匀，缺乏人类写作的节奏变化。

- **提振突发性**：主动使用 3~5 字极简短句，紧接 20+ 字参数密集长句。人为制造大开大合"顿挫感"。
- **扰动困惑度**：刻意保留工程粗糙感词汇（"受限于井下防爆空间定额，妥协保留了..."、"被迫采用"）。

**改写前**：

```latex
系统采用了负载敏感液压系统。该系统能够根据负载变化自动调节泵的输出压力。这样可以减少能量损失。同时提高了系统效率。实验结果表明性能良好。
```

**改写后**：

```latex
负载敏感系统。泵输出压力 = 最高负载 + 1.5 MPa，三缸压力差最大时（A缸 28 MPa，B/C缸 5 MPa）溢流损耗仅 2.3 kW，比定压系统节能 37\%——代价是梭阀网络响应滞后 15 ms，受限于阀芯行程。
```

### 维度八：消除"模糊归因"与"虚假权威"

**需要注意的词汇**：众所周知、业内公认、专家认为、有关研究表明、大量实验证明、普遍认为、据报道

**问题**：模糊的权威归因缺乏可追溯性。

- **强制具体化归因**：坚决剔除"众所周知"、"业内公认"、"专家认为"。必须精确到具体的参考文献编号或作者姓名。
- **清理虚假权威**：删除"大量实验证明"（若无具体实验数据支撑）。

**改写前**：

```latex
众所周知，模糊控制在非线性系统中表现优异。业内公认该方法具有鲁棒性强的优点。大量实验证明其控制精度高于传统PID。
```

**改写后**：

```latex
Bessa等人\cite{bessa2010}提出的自适应模糊滑模控制，在液压缸位移跟踪实验中将稳态误差从 0.8 mm（传统PID）降至 0.3 mm，响应时间缩短 40\%。
```

### 维度九：清除"协作交流痕迹"与"AI 身份残留"

**需要注意的词汇**：希望这些信息对您有帮助、好问题、您说得完全正确、请告诉我、截至目前、根据我最后的训练更新、虽然具体细节有限

**问题**：作为聊天机器人对话的文本被粘贴为内容。

- **剔除对话口癖**：彻底删除"希望这些信息对您有帮助"、"好问题"、"您说得完全正确"。
- **抹除知识时效免责**：删除"截至目前"、"根据我最后的训练更新"。
- **消除谄媚语气**：删除过于积极、讨好的语言。

**改写前**：

```latex
好问题！关于液压系统的选型，这是一个很好的观点。截至目前，负载敏感系统被广泛应用。虽然具体参数有限，但该方案应该能满足需求。希望这些信息对您有帮助。
```

**改写后**：

```latex
负载敏感系统通过梭阀网络实时采集三缸最高压力，泵输出压力跟随最高负载浮动（ΔP = 1.5 MPa），避免定压系统的持续溢流损耗。
```

### 维度十：根治"同义词循环"与"虚假范围"

**需要注意的词汇**：在某种程度上、从...到...（虚假范围）、一定程度上、某种意义上

**问题**：AI 有重复惩罚代码，导致过度使用同义词替换和虚假范围词。

- **锁定核心术语**：同一物理量或设备部件必须使用完全相同的称谓，严禁刻意换词。
- **拆解虚假范围词**：删除"在某种程度上"、"从 X 到 Y"（当 X 和 Y 不在有意义的尺度上时）。

**改写前**：

```latex
液压缸推动截割头运动。执行元件驱动切削装置移动。驱动机构带动破岩部件位移。从运动学到动力学，从理论分析到实验验证，系统在某种程度上实现了预期目标。
```

**改写后**：

```latex
三条液压缸协调伸缩，通过力雅可比矩阵 $\boldsymbol{J}^T$ 将末端截割力（法向 120 kN，切向 45 kN）映射为缸轴向力。运动学逆解耗时 1.8 ms，动力学仿真验证缸压峰值 28.3 MPa，低于额定压力 31.5 MPa。
```

### 维度十一：抑制"过度修饰"与"格式暴露"

**需要注意的词汇**：实现着、体现着、展示着、彰显着（-ing式虚假深度）

**问题**：AI 模仿英文 -ing 句式和过度使用格式装饰。

- **清空表情与特殊装饰**：严禁出现 Emoji 或机械地用粗体强调短语。粗体仅限首次定义术语使用。
- **纠正"着"字虚假深度**：删除模仿英文 -ing 句式的"实现着..."、"体现着..."。
- **内联标题垂直列表**：禁止输出 `\item \textbf{名词：} 解释` 格式的列表。

**改写前**：

```latex
系统正实现着高效的能量转换，体现着先进的控制理念，展示着卓越的工程性能。🚀 \textbf{关键创新点：}采用了负载敏感技术。
```

**改写后**：

```latex
负载敏感系统将溢流损耗从 8.5 kW 降至 2.3 kW，节能 37\%。关键创新点是梭阀网络响应时间压缩至 15 ms，满足 20~50 Hz 振荡频段的压力跟踪需求。
```

### 维度十二：削减"填充短语"与"过度限定"

**需要注意的词汇**：为了实现这一目标、值得注意的是、总的来说、可以说、基本上、似乎、在一定程度上、具有...的能力

**问题**：填充短语和过度限定削弱工程结论的确定性。

- **修剪填充废话**：删除"为了实现这一目标"、"值得注意的是"、"总的来说"。
- **压缩过度限定**：删除"可以说"、"基本上"、"似乎"。工程结论应当是确定性的。
- **拒绝通用积极结尾**：删除段末无实际意义的乐观展望（如"具有广阔的应用前景"）。

**改写前**：

```latex
为了实现这一目标，系统具有快速响应的能力。值得注意的是，控制精度基本上达到了预期。总的来说，该方案似乎可以说具有广阔的应用前景。
```

**改写后**：

```latex
系统阶跃响应时间 1.2 s，稳态误差 ±0.3 mm，满足截割轨迹跟踪精度要求（设计指标 ±0.5 mm）。
```

### 维度十三：避免使用"是"（系动词回避）

- **问题**：AI 喜欢用复杂的结构替代简单的系动词。
- **示例**：❌ "该算法作为提升精度的关键体现" → ✅ "该算法提升了精度"

### 维度十四：打破"否定式排比"

- **问题**："不仅……而且……"或"这不仅仅是关于……，而是……"等结构被过度使用。
- **示例**：❌ "这不仅仅是一次算法升级，而是控制理念的革命" → ✅ "该算法升级改变了控制策略"

### 维度十五：消除"提纲式挑战"部分

- **问题**：AI 生成的文章常包含公式化的"挑战"部分。
- **示例**：❌ "尽管取得了进展，但系统仍面临若干挑战..." → ✅ 直接陈述具体的技术瓶颈（如"高频噪声干扰仍未完全滤除"）。

## Part 2：个性与灵魂（Soul & Personality）

避免 AI 模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。好的学术/工程写作背后有一个真实的工程师。

### 缺乏灵魂的写作迹象（即使技术上"干净"）

- 每个句子长度和结构都相同（缺乏节奏）
- 只有中立报道，没有工程视角的取舍和判断
- 不承认不确定性、复杂工况或现实妥协
- 读起来像产品新闻稿或维基百科文章

### 如何增加工程语调（注入人味）

1. **展现工程妥协（承认复杂性）**：真实的工程没有完美的方案。"虽然提升了精度，但代价是算力开销增加了 30%" 胜过单方面的"该方法表现出显著优越性"。
2. **变化节奏（突发性）**：短促有力的定性结论。然后是参数密集的定量长句。混合使用。
3. **定量参数碾压**：用具体的数字（如 $12.3\%$、$3.8\,\mathrm{s}$）替代"大幅"、"极大地"等虚词。
4. **允许一些"毛刺"**：完美的结构感觉像算法。保留一些工程实践中的"被迫采用"、"受限于硬件"等真实困境描述。

## Part 3：AI 高频敏感词组降重替换字典

### 困境 / 意图类

| ❌ AI 高频词  | ✅ 替换为              |
| ------------- | ---------------------- |
| 双重困境      | 双重问题、局限性       |
| 协同难题      | 协同障碍、控制挑战     |
| 旨在解决/探讨 | 致力于改善、深入分析了 |

### 突破 / 价值支撑类

| ❌ AI 高频词      | ✅ 替换为                                                  |
| ----------------- | ---------------------------------------------------------- |
| 突破...技术瓶颈   | 克服...技术难点                                            |
| 取得了一定进展    | 直接删除或替换为具体科研动作（如"提出了基于小波包的算法"） |
| 提供理论/数据支撑 | 直接删除或改写为"为...奠定基础"                            |

### 通用 AI 高频词与填充短语

| ❌ AI 高频词/短语    | ✅ 替换为                    |
| -------------------- | ---------------------------- |
| 此外 / 值得注意的是  | 直接删除，直述事实           |
| 至关重要 / 关键性的  | 重要、关键（视语境选用）     |
| 深入探讨             | 分析、研究、讨论             |
| 强调 / 突出 / 展示   | 指出、表明、呈现             |
| 复杂/复杂性 / 格局   | 直接删除或用具体技术描述替代 |
| 证明（抽象升华用法） | 表明、验证                   |
| 充满活力的 / 著名的  | 直接删除                     |
| 为了实现这一目标     | 为此                         |
| 由于……的事实         | 因为                         |
| 具有……的能力         | 能够/可以                    |

## Part 4：LaTeX 绝对保护红线

### 禁止全局替换

禁止使用 replaceAll 修改 LaTeX 命令。缺失反斜杠时（如 `includegraphics`），只在当前单词前补全 `\`。

### 排版环境绝对保护

绝不修改、删除或翻译任何以 `\` 开头的命令和环境。特别注意：

- `ctexart` 中文字体设置命令（`\zihao`、`\songti` 等）
- `\begin{}`/`\end{}` 环境声明
- `\label{}`、`\ref{}`、`\includegraphics{}` 等交叉引用命令

### 数学体系免疫

`equation`、`align` 等数学公式环境块内部的字母推导、等式演算 **绝对禁止修改**。

### 禁止内部换行

- 严禁将 `\cite{...}` 或连同前后符号拆分到多行
- 严禁在 `\label{...}` 或 `\includegraphics{...}` 的花括号内部换行
- 百分号必须输出为 `\%`

## Part 5：快速检查清单（Phase 3 自检用）

在交付文本前，进行以下人工/逻辑检查：

### 句式与节奏

- [ ] **连续三个句子长度相同？** 打断其中一个，制造长短句顿挫。
- [ ] **段落以空泛的积极展望结尾？** 删除，陈述完技术事实即结束。
- [ ] **揭示前有破折号（—）？** 删除它，用逗号或句号替代。

### 连接词与逻辑

- [ ] **使用了"此外""然而""值得注意的是"？** 强制删除至少一半。
- [ ] **出现"虽然...但是"模板？** 改为直接陈述双方事实。
- [ ] **三段式列举（A、B和C）？** 改为两项或四项。

### 格式与结构

- [ ] **存在"名词+冒号"的八股列表？** 融合成连贯段落。
- [ ] **章节开头有预告句（"本章将..."）？** 删除，直接进入正文。
- [ ] **图表后有独立解释段（"如图所示，..."）？** 将图表引用自然融入主正文。

### 数据与归因

- [ ] **有无具体数据支撑的"显著提升"？** 补充具体参数或降级表述。
- [ ] **出现"众所周知""专家认为"等模糊归因？** 替换为具体文献引用。
- [ ] **是否体现了工程妥协或现实约束？** 检查是否过于"完美"而缺乏真实感。

### LaTeX 特有检查

- [ ] **`\cite{}` 位置正确？** 紧贴被引文字，在标点内侧，无前置空格。
- [ ] **数学公式格式规范？** 行内用 `$...$`，行间用 `\[...\]` 或 `equation` 环境。
- [ ] **中英文标点混用？** 中文用全角（，。），英文用半角（, .）。
- [ ] **连续括号堆砌？** 如"（XXX）（YYY）"，融合为一个括号或化入正文。
